## LLMへの投入方法

### 1. RAG（検索拡張生成）への投入
```
各JSONファイルをベクトルDBに投入し、鑑定中に関連データを検索して
LLMのコンテキストに追加します。

推奨ベクトルDB: Pinecone / Chroma / Weaviate
推奨埋め込みモデル: text-embedding-ada-002 / multilingual-e5-large
```

### 2. システムプロンプトへの直接投入
```
Markdownファイルをシステムプロンプトのコンテキストとして使用します。
鑑定に必要なカテゴリのMarkdownを選択して投入してください。
```

### 3. RDB（リレーショナルDB）への投入
```
CSVファイルをMySQL/PostgreSQL等のRDBに投入します。
planet, sign, house をキーとしてSQL検索が可能です。
```

### 4. 統合JSONの使用
```
master_astrology_db_v2.json に全データが統合されています。
1ファイルで全カテゴリを参照できます。
```

## 鑑定フロー例

1. **出生データ入力**: 生年月日・出生時刻・出生地
2. **チャート計算**: 各天体の星座・ハウス位置を算出
3. **DB参照**:
   - `sun_in_signs` → 太陽星座の性格
   - `moon_in_signs` → 月星座の感情パターン
   - `asc_in_signs` → ASCの外見・人生テーマ
   - `sun_in_houses` → 太陽ハウスの人生エネルギー
   - `saturn_in_signs` → 土星の試練とカルマ
   - `jupiter_in_houses` → 木星の幸運の領域
4. **アスペクト解釈**: `aspects` データを参照
5. **総合鑑定**: 全データを統合して鑑定文を生成
